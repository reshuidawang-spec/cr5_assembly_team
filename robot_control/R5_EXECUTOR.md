# R5 视觉质量分拣执行器

R5 已接入团队现有接口：

```text
R5_SORT_GOOD_DONE   -> R5
R5_SORT_DEFECT_DONE -> R5
```

两个分支都保持 7 个 Git R5 target 不变，运行时使用 `100 mm` 虚拟
吸盘 TCP。产品 attach 后到释放前保持固定的产品到 TCP 变换，不再单独
修改产品世界位置或姿态。两个分支到 `PLACE_TCP` 后都让 TCP 和产品一起
下降 `26 mm` 到带面，再解除视觉 attach；不改 target 或传送带。

## Defect 分支

Defect 使用 `(195,-45,0)` 度姿态和 `(-0.15,-0.15,0.65)` 私有中转点。
完整产品在整条路径中跟随 R5，到达带面后释放。2026-07-19 刚性同步
回归完成，抓取相对位姿最大误差 `2.78e-16`，R5 回零。

## Good 分支

旧 `(150,45,45)` 直接分支在 good 下降段会发生产品与 R5 Link4 碰撞，
已停用。2026-07-20 对齐路线继续复用 defect 的抓取和竖直抬升，然后显式
执行 `joint1=-121 deg` 相对预转。Git Good APP/TCP 仍为
`(0.65,-1.10,0.62/0.42)`，仅在执行层对坐标副本施加
`X=-10 mm、Y=+20 mm` 偏移：

```text
(195,-45,0) 安全取料
-> 原路竖直抬回 PICK_APP
-> joint1 相对转动 -121 deg
-> (0.630721,-0.931635,0.760000) 高位中转
-> (-143.152079,31.403342,104.057370) deg 预对齐到运行时 Good APP
-> 101 点确定性关节插值到运行时 Good TCP (0.64,-1.08,0.42)
-> 最终 TCP 姿态 (-134.007027,10.545291,79.271417) deg
-> TCP 与产品同步下降 26 mm
-> 带面释放
-> 原路径回零
```

释放段仅允许“产品底面与当前目标带面”接触；R5 本体对带面、产品对其他
环境、产品对 R5、自碰撞和工作空间检查仍保留。新路线先通过 `1054` 个
停止态逐状态检查；释放 yaw 为 `-89.998600 deg`，平行误差
`0.001400 deg`，产品到 TCP 最大误差 `6.11e-16`。

正式单臂 Good 在默认 `50 deg/s`、APP 保持 `0.8 s` 下墙钟 `56.06 s`，
释放 yaw `-89.998160 deg`、平行误差 `0.001840 deg`、抓取相对位姿最大
误差 `1.11e-16`、最大回零误差 `0.004661 deg`，独立 postflight 通过。
新路线随后以 `60 deg/s`、APP 保持 `0.4 s` 完成五臂 Good 7/7 回归，
总墙钟 `230.954 s`，R5 阶段 `45.936 s`；R5 释放 yaw
`-89.998158 deg`、平行误差 `0.001842 deg`、抓取相对位姿最大误差
`1.67e-16`。独立 postflight 最大回零误差为 R5 `0.008217 deg`，五臂
环境/自碰撞均为 none，Runtime 为空。结构化日志为
`data/logs/five_arm_good_yaw_aligned_60_2026-07-20.json`。

该结果证明视觉 attach 后的相对变换固定，不等于真实吸盘抓取或物理负载
验收。禁止在 attach 后单独修改产品世界位置或姿态。

## 命令

```bash
python3 robot_control/run_r5_task.py R5_SORT_GOOD_DONE
python3 robot_control/run_r5_task.py R5_SORT_DEFECT_DONE
```

R5 前置条件是检测产品位于 `(0.15,0.05,0.216)`、完全可见、父节点为
`/Parts`，且 R5 六关节为零。干净场景启动时 Generator 会先隐藏检测产品；
完整五臂流程由 R3 建立可见检测产品，单臂验收则必须先在运行态满足
`30/30` shape 可见。2026-07-20 首次单臂尝试因此被 preflight 拦截且未运动，
恢复运行态可见性后正式运行成功。
