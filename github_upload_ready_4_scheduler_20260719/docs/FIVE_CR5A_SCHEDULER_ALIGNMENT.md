# 4号调度模块五臂新场景对齐说明

## 1. 为什么要改

最新场景统一为 Five CR5A Electrical Control Box Assembly Cell。新场景不再只是“上料、装配、锁付、检测、分拣”的抽象流程，而是明确到电控箱零件、机械臂分工、APP/TCP 目标点和 ROS2 完成信号。

这意味着 4 号调度模块必须使用细粒度任务：装配被拆成箱体、PCB、模块、端子排等步骤，检测由固定相机完成，锁付由 R4 完成，分拣由 R5 完成。

## 2. 新场景下的工序拆分

新调度流程按场景文档拆成以下任务链：

```text
R1 箱体上料
  -> R2 PCB 安装
  -> R3 控制模块安装
  -> R1 端子排安装
  -> R3 转移完整装配体到检测区
  -> CAMERA 固定相机检测
  -> R4 端子螺钉锁付
  -> R5 根据检测结果分拣到合格品/缺陷品传送带
```

其中，`inspect` 负责产生 OK/NG 质量结果，但不会立刻触发分拣。新场景要求 R4 锁付完成后，R5 才能取走产品，因此调度器会先保存检测结果，再在 `screw` 完成后生成 `sort_good` 或 `sort_defect`。

## 3. 资源与区域互斥变化

新版资源包括 R1-R5 五台 CR5A，并把固定相机作为检测资源 `CAMERA` 纳入调度。

关键互斥区域如下：

| 区域 | 作用 | 为什么要锁 |
|---|---|---|
| `assembly_fixture` | 装配夹具 | 箱体、PCB、模块、端子排都在同一装配夹具上完成，不能同时占用 |
| `inspection_platform_area` | 检测/锁付/分拣平台 | R3 放置产品、CAMERA 检测、R4 锁付、R5 取件都依赖该平台 |
| `inspection_screw_area` | 锁付区 | R4 锁付时独占 |
| `good_conveyor_area` | 合格品传送带入口 | R5 放置良品时占用 |
| `defect_conveyor_area` | 缺陷品传送带入口 | R5 放置不良品时占用 |

## 4. 对算法思路的影响

参考研究型 PPT 的表达方式，这次改进可以从“目标”和“解法”两层讲：

目标上，不再只追求简单完工时间，而是要在细化装配流程中兼顾急单响应、交期、共享平台互斥和异常鲁棒性。

解法上，仍然使用动态综合评分法，但调度对象从粗任务升级为细粒度工序 DAG，并加入五臂资源和检测结果延迟触发分拣机制。

## 5. 最新实验结论

在新五臂细化流程下，实验结果如下：

| 方案 | 总完工时间 | 急单响应 | 急单完成 | 加权延期 | 说明 |
|---|---:|---:|---:|---:|---|
| baseline | 195 | 100 | 195 | 525 | 串行执行，最保守 |
| parallel_fifo | 154 | 59 | 154 | 320 | 并行但只按先来先做 |
| proposed | 170 | 3 | 98 | 105 | 动态评分优先保证急单和交期 |
| fault_r4 | 180 | 3 | 122 | 250 | R4 锁付故障后性能下降，但仍可完成 |

与并行 FIFO 相比，本文方案把急单响应从 59 降到 3，急单完成从 154 提前到 98，加权延期从 320 降到 105。代价是总完工时间从 154 增加到 170，说明本文方案更偏向“关键订单和交期可靠性”，不是单纯追求最快完工。

## 6. 可以对负责人说明的话

我根据最新五台 CR5A 电控箱装配场景，对 4 号调度模块做了场景对齐。主要改动是把订单工艺从粗粒度流程细化为箱体上料、PCB 安装、控制模块安装、端子排安装、产品转移、相机检测、R4 锁付和 R5 分拣，并把 R1-R5 和固定相机都纳入调度资源。

同时，我保留了动态综合评分思路，并增加了检测结果延迟触发分拣逻辑：相机先给出 OK/NG，但必须等 R4 锁付完成后，才由 R5 执行良品或缺陷品分拣。这样调度顺序与最新场景文档一致，也更接近真实装配过程。
