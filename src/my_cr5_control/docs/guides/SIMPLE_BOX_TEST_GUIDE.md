# 简化版测试 - 带孔箱体

## 改进内容

### ✅ 已实现
1. **测头碰撞模型**：CR5Robot的init()会自动加载测头
2. **地板**：10m × 10m × 1m，顶面在Z=0
3. **带孔箱体**：用多个Box组合构建
   - 底面：20cm × 20cm × 2cm
   - 4个侧面：壁厚2cm
   - 顶面：环形，中间有8cm直径的孔
   - 孔深度：12cm

### 箱体结构
```
        ┌─────┐ ┌─────┐
        │     │ │     │  ← 顶面（4个矩形条）
        └─────┘ └─────┘
        ╔═══════════════╗
        ║               ║
        ║    ○ 孔      ║  ← 中间有孔
        ║               ║
        ╚═══════════════╝
        └───────────────┘  ← 底面
```

### 测点设计
1. **简单**（0.3）：箱体顶部中心，孔上方5cm
2. **中等**（0.5）：箱体侧面，需要水平姿态
3. **困难**（0.7）：孔内部浅层，深度4cm
4. **极端**（0.9）：孔内部深层，深度10cm

## 运行测试


## 在RViz中查看

测试运行时，在RViz中可以看到：
- **绿色**：机械臂
- **橙色/红色**：测头（附加在Link6上）
- **灰色**：箱体（由多个Box组成）
- **灰色平面**：地板

你应该能清楚地看到箱体顶部的孔，以及测头进入孔内部的运动。

## 预期结果

### 简单场景（孔上方）
- 所有规划器都应成功
- RRTConnect应该最快

### 中等场景（侧面）
- 需要特定姿态
- BIT*可能开始显示优势

### 困难场景（孔内浅层）
- 狭窄空间约束
- BIT*应该表现更好

### 极端场景（孔内深层）
- 极度狭窄
- BIT*的批量采样应发挥作用

## 如果仍然失败

如果测试仍然失败（规划器无法找到路径），可能的原因：
1. 测点位置仍然不合理
2. 箱体的孔太小
3. 需要一个真实的STL模型

**如果需要，请提供一个带孔的STL模型**，我会修改代码直接加载STL文件。

STL模型要求：
- 格式：二进制STL
- 单位：米（或毫米，我会转换）
- 特征：顶部有一个圆孔或方孔
- 尺寸：建议20-30cm的箱体，孔直径5-10cm
