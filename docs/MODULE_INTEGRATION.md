# 成品倒推与模块集成方案

## 1. 最终成品定义

本项目最终成品是一套能够连接 CoppeliaSim 仿真场景的软件系统，而不是单独的场景或脚本。目标闭环如下：

```text
软件端输入订单
    ↓
订单拆解模块生成工序任务
    ↓
调度模块选择任务和机械臂
    ↓
机械臂控制模块执行动作
    ↓
CoppeliaSim 场景中机械臂运动
    ↓
机械臂返回任务状态和检测结果
    ↓
软件界面刷新任务进度和效能指标
```

核心演示流程：

```text
A/B/C 型柜订单输入
    ↓
R1 上料定位
    ↓
R2 元件装配
    ↓
R3 螺丝锁付
    ↓
R3 质量检测
    ↓
R4 根据检测结果进行良品/不良品分拣
    ↓
可选：R4 将不良品送入返修拆解区
```

---

## 2. 模块边界

### 2.1 软件集成模块

职责：

- 提供订单输入入口；
- 调用订单拆解与调度模块；
- 调用机械臂执行接口；
- 显示 CoppeliaSim 连接状态；
- 显示任务队列和机械臂状态；
- 记录日志并展示效能指标。

不负责：

- 不直接编写机械臂轨迹；
- 不直接定义新的任务格式；
- 不直接修改 CoppeliaSim 场景对象。

---

### 2.2 CoppeliaSim 场景模块

职责：

- 搭建低压配电柜柔性装配工作空间；
- 放置 R1、R2、R3、R4 四台机械臂；
- 设置上料区、元件料仓区、中央装配区、锁付区、检测区、良品区、不良品区、返修区；
- 给所有关键点位和对象统一命名；
- 输出点位表，供调度和控制模块调用。

不负责：

- 不写订单拆解逻辑；
- 不直接决定任务调度顺序；
- 不私自更改接口命名。

---

### 2.3 机械臂控制模块

职责：

- 接收调度模块下发的任务指令；
- 将任务指令转换为 CoppeliaSim 中的机械臂动作；
- 实现夹爪抓取、元件放置、螺丝锁付、检测动作、良品/不良品分拣；
- 任务结束后返回统一状态。

统一接口：

```python
def execute_task(task_command: dict) -> dict:
    pass
```

不负责：

- 不判断订单优先级；
- 不生成任务队列；
- 不决定哪个任务先执行。

---

### 2.4 订单拆解与调度模块

职责：

- 读取订单；
- 根据产品类型生成工序任务；
- 根据前置约束、优先级、机械臂状态和区域锁进行任务分配；
- 接收 R3 检测结果；
- 根据检测结果生成 R4 分拣任务；
- 必要时生成不良品返修拆解任务；
- 输出日志和调度结果。

不负责：

- 不直接操作 CoppeliaSim；
- 不直接控制机械臂关节；
- 不绘制软件界面。

---

## 3. 关键接口

模块之间只通过以下接口交互：

| 接口 | 发送方 | 接收方 | 作用 |
|---|---|---|---|
| 订单接口 | 软件模块 | 调度模块 | 输入订单信息 |
| 任务接口 | 调度模块 | 机械臂控制模块 | 下发动作任务 |
| 点位接口 | 场景模块 | 调度/控制模块 | 统一场景目标点 |
| 状态接口 | 机械臂控制模块 | 调度/软件模块 | 返回执行结果 |
| 日志接口 | 调度模块 | 软件/数据模块 | 输出实验数据 |

如果某个模块还没完成，必须先提供 Mock 版本，保证主系统不断线。

## 4. 最小闭环版本

### V0.1 控制台闭环

目标：

```text
订单输入 → 任务生成 → Mock 机械臂执行 → 输出日志
```

验收：

- 输入 A 型柜订单；
- 生成 R1、R2、R3、R4 的任务；
- R3 检测结果可以手动设为 OK 或 NG；
- R4 能根据 OK/NG 生成对应分拣任务。

---

### V0.2 CoppeliaSim 单任务联动

目标：

```text
软件/脚本发出一个任务 → CoppeliaSim 中一台机械臂执行动作 → 返回完成
```

验收：

- R1 能执行上料动作；
- 或 R4 能执行分拣动作；
- 执行完成后返回 `finished` 状态。

---

### V0.3 完整产品流程

目标：

```text
A 型柜订单 → R1 上料 → R2 装配 → R3 锁付 → R3 检测 → R4 分拣
```

验收：

- R3 检测为 OK 时，R4 去良品区；
- R3 检测为 NG 时，R4 去不良品区；
- 软件显示每个任务状态。

---

### V1.0 比赛展示版

目标：

```text
A/B/C 动态订单 + 急单插入 + 区域避碰 + 检测分拣 + 数据看板
```

验收：

- 软件界面可输入订单；
- CoppeliaSim 能展示四机械臂协同；
- 调度算法能处理急单或异常；
- R4 能完成质量分拣；
- 输出总完成时间、机械臂利用率、等待时间、冲突次数和分拣响应时间。

---

## 5. 防止最后无法整合的要求

1. 第一周冻结接口，不允许每个人单独定义数据格式；
2. 所有点位必须以 `P_` 开头并写入 `configs/points.yaml`；
3. 所有任务动作必须从统一动作集合中选择，如 `feed`、`assemble`、`screw`、`inspect`、`sort_good`、`sort_defect`、`rework`；
4. 所有机械臂执行结果必须返回 `task_id`、`robot_id`、`status`、`start_time`、`end_time`；
5. 每周至少集成一次，不允许最后一周才合并；
6. 真实模块未完成时必须提供 Mock 模块；
7. 软件集成模块从第一周开始维护主入口，不等其他模块全部完成。
