# 两个月并行开发计划（四臂方案草案，尚未与五臂场景整合）

> 状态说明（2026-07-17）：本文件记录的是四臂低压配电柜方案与阶段性目标，并非当前已实现进度。仓库实际提交的 CoppeliaSim 场景采用五台 CR5A、小型电控箱工艺，详见 `SCENE_BUILDING_GUIDE.md`；Python 主控仍运行 Mock 模式。以 README 的“当前工作线”表为状态基准。

## 项目题目

**工序自适，群臂协同——面向多工艺柔性产线的多机械臂自主调度与效能优化系统**

## 成品目标

最终作品不是单一的 CoppeliaSim 场景，也不是单独的调度脚本，而是一套可演示的软件系统：

```text
软件端输入订单
    ↓
订单拆解与任务调度
    ↓
机械臂控制模块接收任务
    ↓
CoppeliaSim 场景中执行上料、装配、锁付、检测、分拣
    ↓
状态反馈到软件界面
    ↓
输出任务日志和效能指标
```

项目采用 R1-R4 四机械臂方案：

```text
R1：上料定位
R2：元件装配
R3：螺丝锁付与质量检测
R4：根据 R3 的 OK/NG 检测结果进行良品/不良品分拣，可选返修拆解
```

R1-R3 是基础装配闭环，R4 是质量分拣闭环。两个月内优先保证 R4 完成良品/不良品分流，返修拆解作为时间充足后的增强功能。

---

## 总体策略

本项目采用“双系统并行 + 持续集成”的开发方式：

1. **算法模拟系统**：不依赖完整仿真场景，优先完成订单解析、任务分解、动态调度、检测结果分支、日志统计和对比实验。
2. **CoppeliaSim 仿真系统**：先搭建极简场景，再逐步替换为配电柜、元件、螺丝点、检测区、良品区、不良品区等真实模型。
3. **软件集成系统**：从第一周开始设计主控程序和界面框架，先用 Mock 模块跑通，再逐步接入真实调度、真实仿真控制和数据看板。

核心原则：

- 先统一接口，再并行开发；
- 先跑通极简闭环，再美化场景；
- 先用 Mock 模块保证联动，再替换真实模块；
- 软件集成同学不能最后才开始，而要从第一周就定义数据流和主入口；
- R4 分拣作为主线增强功能，优先实现 OK/NG 分流；
- R4 返修拆解作为可选功能，不应影响基础演示交付。

## 集成节点

### 第 1 个集成点：第 1 周末

目标：接口冻结与最小 Mock 闭环。

验收标准：

- 有统一订单格式；
- 有统一任务格式；
- 有统一点位命名；
- 有统一状态反馈格式；
- 软件或主控脚本能调用 Mock 调度和 Mock 机械臂执行器。

### 第 2 个集成点：第 2 周末

目标：算法模拟系统能独立运行。

验收标准：

- 输入订单 JSON；
- 输出任务队列；
- 输出调度日志；
- 能统计总完成时间；
- 能根据 R3 的模拟检测结果生成唯一的 R4 分拣任务。

### 第 3 个集成点：第 4 周末

目标：仿真系统能完成一轮固定流程演示。

验收标准：

- R1-R3 能完成上料、装配、锁付/检测基础流程；
- R4 能完成良品/不良品分拣动作；
- 共享区域不会发生明显穿模或碰撞；
- 软件界面能显示任务队列和执行状态。

### 第 4 个集成点：第 6 周末

目标：算法结果驱动仿真演示。

验收标准：

- 调度模块输出机械臂任务；
- 仿真模块根据任务执行动作；
- 能演示急单插入或故障重分配；
- 能演示 R3 检测不良后 R4 分拣到不良品区；
- 能导出实验数据。

### 第 5 个集成点：第 8 周

目标：形成可提交作品。

验收标准：

- README 完整；
- 软件或主控程序可运行；
- 仿真视频可播放；
- 技术报告、PPT、实验图表齐全；
- 有备用演示方案；
- R4 返修拆解若未完成，应作为拓展功能写入报告而非强行演示。
