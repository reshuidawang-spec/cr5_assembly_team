# 两个月并行开发计划

## 项目题目

**工序自适，群臂协同——面向多工艺柔性产线的多机械臂自主调度与效能优化系统**

## 总体策略

本项目采用“双系统并行 + 后期集成”的开发方式：

1. **算法模拟系统**：不依赖完整仿真场景，优先完成订单解析、任务分解、动态调度、日志统计和对比实验。
2. **仿真演示系统**：先搭建极简场景，再逐步替换为配电柜、元件、螺丝、检测区等更真实的模型。

核心原则：

- 先统一接口，再并行开发；
- 先跑通极简闭环，再美化场景；
- 先保证演示稳定，再增加算法亮点；
- 不把所有工作都卡在仿真场景完成之后。

## 每名成员的最低交付物

### 1号：项目负责人

- 课题简介；
- 系统架构图；
- 技术方案报告；
- 答辩 PPT；
- 团队进度表；
- 最终提交清单。

### 2号：仿真场景

- 上料区、装配区、锁付区、检测区；
- 配电柜与元件模型；
- 可录制的演示视角；
- 3–5 分钟仿真演示视频。

### 3号：动作与路径

- `move_home()`；
- `move_to_pick()`；
- `move_to_place()`；
- `move_to_screw()`；
- `move_to_inspect()`；
- 区域锁避碰逻辑；
- 与调度输出的接口适配。

### 4号：调度算法

- 订单输入格式；
- 任务队列生成；
- 固定顺序 baseline；
- 动态优先级调度；
- 急单插入；
- 故障重分配；
- 调度日志输出。

### 5号：数据与展示

- 任务日志 CSV；
- 总完成时间对比；
- 机械臂利用率统计；
- 等待时间统计；
- 冲突次数统计；
- 图表与数据看板；
- 视频剪辑与展示材料。

## 集成节点

### 第 1 个集成点：第 2 周末

目标：算法模拟系统能独立运行。

验收标准：

- 输入订单 JSON；
- 输出任务队列；
- 输出调度日志；
- 能统计总完成时间。

### 第 2 个集成点：第 4 周末

目标：仿真系统能完成一轮固定流程演示。

验收标准：

- 三台机械臂能够按任务顺序动作；
- 至少包含上料、装配、锁付/检测三类工艺；
- 共享区域不会发生明显穿模或碰撞。

### 第 3 个集成点：第 6 周末

目标：算法结果驱动仿真演示。

验收标准：

- 调度模块输出机械臂任务；
- 仿真模块根据任务执行动作；
- 能演示急单插入或故障重分配；
- 能导出实验数据。

### 第 4 个集成点：第 8 周

目标：形成可提交作品。

验收标准：

- README 完整；
- 代码可运行；
- 仿真视频可播放；
- 技术报告、PPT、实验图表齐全；
- 有备用演示方案。
