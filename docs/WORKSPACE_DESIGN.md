# 工作空间设计与零部件清单（四臂方案草案）

> 本文是低压配电柜四臂方案的设计/采购草案。当前仓库实际的 CoppeliaSim 场景是五臂 CR5A 小型电控箱单元，布局和目标点以 `SCENE_BUILDING_GUIDE.md` 与 `scenes/main_cell_generator.lua` 为准；两者尚未完成统一。

面向低压配电柜柔性装配的 **四机械臂协同工作单元**。

R1-R3 完成上料、装配、锁付和检测的基础装配闭环；R4 根据 R3 的检测结果完成良品/不良品分拣，并在时间充足时扩展为不良品返修拆解。

---

## 1. 总体布局

推荐采用：

```text
四机械臂 + 中央装配工位 + 上料/料仓/检测/分拣/返修区域 + 共享避碰区
```

布局逻辑如下：

```text
┌────────────────────────────────────────────────────────────┐
│                    柔性装配工作空间                         │
│                                                            │
│  上料区 / 原料区                  元件料仓区                 │
│  ┌────────────┐                ┌────────────┐              │
│  │ 柜体底板   │                │ 电气元件   │              │
│  │ 安装板     │                │ 端子/模块  │              │
│  └────────────┘                └────────────┘              │
│       ↑                              ↑                     │
│     R1 机械臂                      R2 机械臂                │
│   上料定位臂                      元件装配臂                 │
│                                                            │
│                 ┌──────────────────────┐                   │
│                 │    中央装配工位       │                  │
│                 │   配电柜安装板        │                  │
│                 │   共享作业区域        │                  │
│                 └──────────────────────┘                   │
│                           ↑                                │
│                        R3 机械臂                            │
│                  锁螺丝 / 检测机械臂                         │
│                                                            │
│         良品区              不良品区              返修区      │
│      ┌────────┐          ┌────────┐          ┌────────┐     │
│      │ GOOD   │          │ DEFECT │          │ REWORK │     │
│      └────────┘          └────────┘          └────────┘     │
│                  ↑                                         │
│               R4 机械臂                                    │
│        分拣 / 下料 / 可选返修拆解                            │
└────────────────────────────────────────────────────────────┘
```

R4 建议放置在检测区、成品下料区和返修区之间，确保其工作半径能覆盖 `P_GOOD_01`、`P_DEFECT_01` 和 `P_REWORK_01`。

## 2. 四台机械臂分工

| 机械臂 | 推荐位置 | 职责 | 推荐末端 |
|--------|----------|------|----------|
| R1 | 左侧上料区附近 | 上料、搬运、安装板定位 | 夹爪 / 吸盘 |
| R2 | 右侧料仓区附近 | 元件抓取、元件装配 | 小型夹爪 |
| R3 | 中央装配区下方或正前方 | 螺丝锁付、视觉检测、返回 OK/NG | 电批模型 + 相机模型 |
| R4 | 检测区与下料区旁 | 良品/不良品分拣，空闲时可选返修拆解 | 夹爪 / 吸盘 |

R4 不建议一开始承担复杂拆解任务。开发优先级应为：

```text
第一阶段：R4 到点分拣
第二阶段：R4 抓取并移动产品模型
第三阶段：R4 将不良品送入返修区
第四阶段：R4 执行拆解动作或返修预处理
```

## 3. 功能区域划分

### 3.1 上料区 `feed_area`

- 配电柜安装底板，A/B/C 型三种；
- 料盘、定位销、挡块；
- 待装配工件托盘；
- 推荐点位：`P_FEED_01`。

### 3.2 元件料仓区 `component_area`

| 元件 | 表示方式 |
|------|----------|
| 断路器 | 长方体模块 |
| 接触器 | 较大长方体模块 |
| 继电器 | 小方块模块 |
| 端子排 | 长条形模块 |
| DIN 导轨 | 细长金属条 |
| 螺丝盒 | 盒子模型 |

前期可以用 3D 打印件、亚克力块或简单几何体代替真实电气元件。

### 3.3 中央装配工位 `assembly_area`

中央装配工位是协同调度和避碰展示的核心区域。

| 子区 | 可进入机械臂 | 用途 |
|------|-------------|------|
| 装配区 A | R1 | 放置安装板 |
| 装配区 B | R2 | 放置元件 |
| 锁付区 C | R3 | 螺丝锁付 |
| 检测区 D | R3 | 检测并返回 OK/NG |
| 共享区 S | R1/R2/R3/R4 需申请区域锁 | 过渡和协同 |

推荐点位：

| 点位 | 含义 | 机械臂 |
|---|---|---|
| `P_ASSEMBLY_01` | 元件安装点 1 | R2 |
| `P_SCREW_01` | 螺丝锁付点 1 | R3 |
| `P_INSPECT_01` | 检测点 1 | R3 |

### 3.4 锁付检测区 `inspect_area`

- 螺丝孔位标记点；
- 电批末端模型；
- 相机检测模型；
- 合格/不合格状态标识；
- R3 检测结果作为 R4 分拣任务触发条件。

检测结果建议统一为：

| 值 | 含义 | 后续动作 |
|---|---|---|
| `OK` | 合格 | R4 执行 `sort_good` |
| `NG` | 不合格 | R4 执行 `sort_defect` |
| `UNKNOWN` | 检测异常 | 复检或人工复核，可选 |

### 3.5 良品区 `good_area`

- 成品托盘；
- 合格品下料位；
- 推荐点位：`P_GOOD_01`；
- 执行动作：`sort_good`。

### 3.6 不良品区 `defect_area`

- 不良品托盘；
- 不合格标识；
- 推荐点位：`P_DEFECT_01`；
- 执行动作：`sort_defect`。

### 3.7 返修区 `rework_area`，可选

- 返修暂存托盘；
- 拆解/复检工位；
- 推荐点位：`P_REWORK_01`；
- 执行动作：`rework`。

返修拆解是拓展功能，不应影响基础演示。若时间不足，只展示 R4 将不良品分拣到不良品区即可。

## 4. 产品类型定义（A/B/C 型柜）

| 型号 | 元件数量 | 螺丝数量 | 检测点数 | 分拣任务 | 工艺复杂度 |
|------|---------|----------|---------|----------|-----------|
| A 型柜 | 3 | 6 | 2 | 1 | 简单 |
| B 型柜 | 5 | 10 | 3 | 1 | 中等 |
| C 型柜 | 6 | 12 | 4 | 1 | 较复杂 |

标准工艺链：

```text
上料 feed → 装配 assemble → 锁付 screw → 检测 inspect → 分拣 sort_good / sort_defect
```

拓展工艺链：

```text
检测 NG → 分拣到不良品区 → 可选 rework 返修拆解
```

## 6. 最小可行版本（仿真优先）

| 必须有 | 可以简化 |
|--------|---------|
| 4 台机械臂，至少仿真中完整展示 R1-R4 | 螺丝用点位停留表示，不用真实拧紧 |
| 1 个中央装配工位 | 检测用 OK/NG 状态判断代替真实视觉 |
| 上料区 + 料仓区 + 锁付检测区 + 良品/不良品区 | 线束用未来拓展表示 |
| A/B/C 三种产品任务 | 输送带用静态模型代替 |
| 4 类工艺：上料、装配、锁付/检测、分拣 | 不做复杂物理抓取仿真 |
| R3 检测结果触发 R4 分拣 | 返修拆解作为可选功能 |

## 7. 实施顺序

```text
第 1 步：确定工作台尺寸 + R1-R4 四台机械臂位置
第 2 步：建立中央装配工位模型
第 3 步：放置上料区、元件料仓区、检测区、良品区、不良品区
第 4 步：定义每个区域的关键坐标点，写入 configs/points.yaml
第 5 步：建立 A/B/C 三类产品任务定义
第 6 步：实现单机械臂动作模板
第 7 步：实现 R1-R3 基础装配流程
第 8 步：实现 R3 检测 OK/NG 返回
第 9 步：实现 R4 良品/不良品分拣
第 10 步：加入共享区域锁避碰、急单插入和故障重分配
第 11 步：有余力再加入 R4 不良品返修拆解
第 12 步：美化场景并录制视频
```

核心思路：先把工作空间抽象成「区域 + 点位 + 任务 + 检测结果」，再逐步美化。R4 的返修拆解应作为增强演示，不影响基础闭环交付。
